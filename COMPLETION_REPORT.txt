================================================================================
MODULE DETAILS
================================================================================

1. weight_sweep_framework.py
   Purpose: Deterministic hyperparameter optimization
   Key Classes:
   - WeightSweepOptimizer: Main sweep engine
   - SweepResult: Result dataclass
   Key Methods:
   - generate_sweep_configs(): Create parameter grid
   - evaluate_config(): Test single config
   - run_sweep(): Run full sweep
   - _compute_parameter_importance(): Attribute metric impact
   Scoring: 50% recall + 30% precision + 15% remediation + 5% latency

2. two_stage_retrieval.py
   Purpose: High-precision incident retrieval with controlled recall
   Key Classes:
   - TwoStageRetrieval: Main retrieval policy
   Key Methods:
   - select_top_k(): Two-stage filtering
   - is_family_safe(): Family safety checks (4 rules)
   - compute_evidence_score(): Evidence aggregation
   - deduplicate_families(): Remove same-family clustering
   Architecture:
   - Stage A: Similarity ≥ 0.60 + family safety → 3-5 results
   - Stage B: Similarity ≥ 0.50 if A < 3 → up to 5 total

3. remediation_optimizer.py
   Purpose: Bayesian-posterior confidence for remediation actions
   Key Classes:
   - RemediationOptimizer: Main optimizer
   Key Methods:
   - score_remediation_family(): Score by family success
   - rank_remediations(): Aggregate and rank top-3
   - confidence_for_action(): Compute confidence
   Confidence Formula:
   - Prior: Beta(2,2) centered at 0.5
   - Posterior: (alpha + successes) / (alpha + beta + total)
   - Final: 40% empirical + 60% posterior, scaled by credibility
   - Capped: [0.05, 0.99]

4. family_representation.py
   Purpose: Prevent same-family clustering in top-5
   Key Classes:
   - FamilyRepresentation: Family grouping and ranking
   Key Methods:
   - family_id_from_incident(): Extract family suffix
   - diversify_by_family(): Diversify to target count
   - compute_family_posterior(): Family statistics
   Family Extraction: "INC-X-5" → family "5" (suffix)

5. decoy_suppression.py
   Purpose: Detect and suppress low-evidence false positives
   Key Classes:
   - DecoySuppressionEngine: Main suppression engine
   - SuppressionPolicy: Policy configuration
   Key Methods:
   - is_likely_decoy(): Check evidence completeness
   - build_suppression_policy(): Create policy from evidence
   - apply_suppression(): Apply policy to matches
   Decoy Detection: ANY missing (deploy + metric + trace/log)
   Suppression: Confidence × 0.60, similarity capped at 0.45
